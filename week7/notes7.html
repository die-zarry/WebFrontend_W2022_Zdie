Client and Servers

Internet can be separated into two parts: clients and servers. A client, such as a web browser, will make a request for a web pages from a server, which processes the request and sends back a response to the client.

JavaScript was at first a clent side language; which means it operated in the browser and could modify web pages sent to the browser form the the server.However, with Ajax, JavaScript can send request to the sever.These resources requested are usually JSON data, fragments of text or HTML rather than a whole web page.

To use Ajax a server is required to respond to those requests or a third party using corss orifin ressourse sharing. CORS

The fetch API

API are software that enable applications to communicate.

The fetch API or the request and response data asynchronously across a network.The fetch() method has only one mandatory argument-the URL of the resource you wish to fetch. It returns a promise which will be resolved when a responce is recieved from the url. A catch statement is sometimes used to handle any errors occurs a long the way

The responce Interface

When the promise is fullfiled, an object is returned and the fetch API provide a responce interface to deal with that. Each object responce has an ok property and method to check if the responce was succesfull.An if block can be used to check if the request was successful, and throw an error otherwise. Some other property of the respnce obhect are: Headers, url, redirected and type. The methods are: redirect, The text() method, the blob() and the json() method
